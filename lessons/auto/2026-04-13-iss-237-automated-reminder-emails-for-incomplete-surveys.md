# Lessons Learned: Automated reminder emails for incomplete surveys

## What Worked Well
- Building on existing email invitation infrastructure (ISS-232/233/236) made the reminder feature straightforward to layer on top
- The existing `send_invitation()` service function was reusable for reminders by passing `invitation_type='reminder'`
- Separating the recipient-filtering logic (days_since_invite, max_reminders, bounced/completed exclusions) into a dedicated `send_reminders()` service function kept the API router thin and the logic testable
- Frontend dialog pattern (preview count → configure → confirm → send) matched user mental model and was easy to test with MSW

## What Was Challenging
- Alembic autogenerate silently drops `server_default` for integer columns — had to manually author migration 0025 to ensure `reminder_count` defaulted to 0 correctly in DDL
- Missing model import in `alembic/env.py` causes silent migration gaps; required adding `EmailInvitation` import before any alembic command
- Docker-based test environment required explicit `postgresql+asyncpg://` DATABASE_URL scheme — plain `postgresql://` fails with a confusing error rather than a clear driver mismatch message

## Key Technical Insights
1. Always manually author migrations for columns with defaults rather than relying on autogenerate — use `sa.Column('reminder_count', sa.Integer(), server_default=sa.text('0'), nullable=False)` explicitly.
2. Run an import smoke-test before every alembic command: `python -c "from app.models.email_invitation import EmailInvitation"` to catch broken imports before migration failures.
3. New SQLAlchemy models must be imported in both `alembic/env.py` and `app/models/__init__.py` or alembic will silently skip the table in autogenerate output.
4. Function-scoped async fixtures are required for all pytest-asyncio tests using asyncpg — session-scoped async engine fixtures cause event loop mismatch errors with no clean workaround.
5. Do not use `passlib` CryptContext with bcrypt >= 4.x in any new service code; use direct `bcrypt` calls instead.

## Reusable Patterns
- **Batch operation endpoint pattern**: accept optional filter params (days_since_invite, max_reminders), query eligible records, process each, return `{sent, skipped, failed}` summary — reusable for any bulk-action endpoint.
- **Reminder dialog UX pattern**: fetch preview count of eligible recipients before confirming send; surface the count in the dialog so users understand scope before committing.
- **Migration authoring checklist**: (1) import model in env.py and __init__.py, (2) run import smoke-test, (3) author migration manually with explicit column defaults, (4) inspect generated script before applying.
- **Docker test invocation**: `docker compose up -d postgres && docker run --rm --network host -e DATABASE_URL="postgresql+asyncpg://survey:survey@localhost:5432/survey_test" -e JWT_SECRET=testsecret -v $(pwd)/backend:/app survey_tool-backend:latest python -m pytest tests/ -q`

## Gotchas and Pitfalls
- **Alembic autogenerate + server_default**: autogenerate silently omits `server_default` and `onupdate` directives — never trust autogenerated output for columns with defaults; always inspect and patch manually.
- **alembic/env.py import gap**: if the model is not imported in env.py before running `alembic revision --autogenerate`, the new table/column is invisible to autogenerate with no error or warning.
- **asyncpg scheme requirement**: DATABASE_URL must use `postgresql+asyncpg://` — using `postgresql://` in Docker tests raises a confusing connection error unrelated to the actual problem.
- **pytest-asyncio fixture scope**: async engine and session fixtures must be `scope='function'`; session-scoped async fixtures cause event loop conflicts with asyncpg that cannot be worked around without changing scope.
- **passlib + bcrypt >= 4.x**: incompatible — any new service code that needs password hashing must use direct `bcrypt` calls, not `passlib.CryptContext`.
- **reminder_count increment timing**: increment `reminder_count` only after a successful send, not before, to avoid inflating the count on transient failures.
