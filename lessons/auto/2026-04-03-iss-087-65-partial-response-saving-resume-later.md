# Lessons Learned: 6.5: Partial Response Saving (Resume Later)

## What Worked Well
- Differentiating partial save vs. completion within the existing PATCH endpoint by checking whether `status='complete'` was provided kept the API surface minimal and backward-compatible
- Upsert via SQLAlchemy ON CONFLICT DO UPDATE cleanly handled repeated partial saves without duplicating answer rows
- Reusing existing async pytest fixture patterns from prior tickets (function-scoped engine/session) avoided event loop mismatch issues from the start
- Extending `ResponseResponse` with an `answers` array field served both the GET resume endpoint and the PATCH partial-save response without requiring a separate schema

## What Was Challenging
- Confirming that the `(response_id, question_id)` unique constraint existed at the database level (not just in the ORM) before implementing ON CONFLICT DO UPDATE — autogenerate cannot be trusted to produce or preserve this correctly
- Ensuring that completion validation was entirely bypassed on partial saves while still being triggered on PATCH with `status='complete'`, given both paths share the same endpoint
- Eager-loading answers in `get_response_with_answers()` without accidentally triggering lazy-load errors in an async SQLAlchemy session

## Key Technical Insights
1. ON CONFLICT DO UPDATE in PostgreSQL requires the unique constraint to exist at the database level. Verify with `\d response_answers` or inspect migration history before relying on it in service code.
2. Async SQLAlchemy fixtures must be `scope='function'`. Session-scoped async fixtures cause event loop mismatch errors with asyncpg under pytest-asyncio — this is non-negotiable.
3. The DATABASE_URL in tests must use the `postgresql+asyncpg://` scheme. The default `postgresql://` scheme will fail silently with an async engine.
4. Alembic autogenerate silently drops `server_default` and `onupdate` directives on timestamp columns. Any migration touching tables with these columns must be manually authored and inspected.
5. Partial save must not invoke any completion or required-field validation. These two code paths must be kept strictly separate even when routed through the same HTTP endpoint.
6. Run a Python import smoke-test (`python -c 'from app.models.response_answer import ResponseAnswer'`) before any `alembic` command to catch broken imports as clean tracebacks rather than cryptic migration errors.

## Reusable Patterns
- **Upsert pattern**: Use SQLAlchemy `insert().on_conflict_do_update(index_elements=[...], set_={...})` for answer merging; requires the unique constraint to exist at the DB level.
- **Endpoint routing by payload**: Check a sentinel field (e.g., `status`) in the request body to route to different service functions within a single PATCH endpoint rather than adding new endpoints.
- **Resume GET endpoint**: Return the full response with eagerly loaded answers so the frontend can reconstruct form state without additional round-trips.
- **Function-scoped async fixtures**: Always decorate with `@pytest_asyncio.fixture(scope='function')` for engine, session, and client fixtures when using asyncpg.
- **Import smoke-test**: `python -c 'from app.models.<model> import <Model>'` as a pre-flight check before every alembic command.
- **localStorage keyed by survey_id**: Frontend stores `response_id` under a key like `survey_response_<survey_id>` to enable transparent resume on revisit.

## Files to Review for Similar Tasks
- `backend/app/api/responses.py` — PATCH/GET endpoint structure and payload-based routing logic
- `backend/app/services/response_service.py` — `save_partial_response()` and `get_response_with_answers()` implementations
- `backend/app/schemas/response.py` — `ResponseResponse` schema with embedded `answers` array
- `backend/tests/test_responses.py` — async fixture patterns and partial-save test cases
- Alembic migration that adds the `(response_id, question_id)` unique constraint — verify it was manually authored and not autogenerated

## Gotchas and Pitfalls
- **Silent migration gaps**: New models must be imported in both `alembic/env.py` and `app/models/__init__.py` before running any alembic command, or migrations will silently skip them.
- **Autogenerate drops timestamp directives**: Never trust autogenerate output for tables with `server_default` or `onupdate` — always manually author and inspect these migrations.
- **Unique constraint must exist at DB level**: The ORM `UniqueConstraint` declaration alone is not enough for ON CONFLICT DO UPDATE. Confirm the constraint is present in the actual database schema.
- **Do not use passlib CryptContext**: bcrypt 5.0.0 is installed; passlib 1.7.x raises `AttributeError` at runtime. Use bcrypt directly anywhere password hashing is needed.
- **Lazy-load errors in async sessions**: Use `selectinload` or `joinedload` explicitly when returning related objects (e.g., answers) from an async service function — async sessions do not support implicit lazy loading.
- **Partial save must not change status**: After a partial save the response status must remain `incomplete`. Any code path that sets status must be guarded to only run on explicit completion requests.
